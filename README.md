## 왜 이 프로젝트가 중요한가

이 프로젝트는 단순 군집화 과제가 아니라, 은행의 수익 구조 문제를 데이터로 해석하고 전략으로 연결한 사례입니다.

프로젝트 당시 팀은 BNK부산은행이 다음과 같은 구조적 과제를 안고 있다고 판단했습니다.

- 이자이익 의존도가 매우 높아 외부 환경 변화에 취약할 수 있음
- 비이자이익 상품 미이용 고객 비중이 높아 성장 여지가 큼
- 고객을 하나의 기준으로만 분류하면 실제 수익 전략으로 연결하기 어려움

따라서 분석의 출발점을 `변수 선택`이 아니라 `문제 정의`에 두었습니다.  
핵심 질문은 아래 두 가지였습니다.

1. 어떤 고객이 안정적인 이자이익에 기여하는가?
2. 어떤 고객이 비이자이익 확대의 잠재력이 큰가?

이 질문에 답하기 위해 고객을 하나의 축으로 묶지 않고, `이자이익`과 `비이자이익`을 분리한 **Two-track 분석 프레임**을 설계했습니다.

## 핵심 성과

- BNK 고객을 수익 기여 구조 기준으로 재해석하는 세분화 프레임 제안
- 비이자이익 관점에서 `Basic → Standard → Premium → VIP` 성장 로드맵 도출
- 이자이익 관점에서 대출 상품별 잠재고객 프로필과 관리 방향 정리
- 군집 결과를 마케팅 문장이 아닌 실행 전략으로 번역
- 발제사 실무진으로부터 "실제 현업에서도 고려하는 방향"이라는 피드백 확보

## 문제 정의 방식

많은 해커톤 프로젝트가 주어진 변수를 모두 넣고 모델을 돌린 뒤 해석을 붙이는 방식으로 진행됩니다. 이 프로젝트는 그 반대로 접근했습니다.

먼저 사업 문제를 정리했습니다.

- 은행 수익의 안정성은 이자이익이 담당한다.
- 은행 수익의 성장성은 비이자이익이 담당한다.
- 두 수익원은 고객 행동, 활용 상품, 관리 목표가 다르다.

이 판단을 바탕으로 분석 구조를 아래처럼 분리했습니다.

### 1. 비이자이익 관점

- 핵심 변수: 펀드잔액, 방카슈랑스잔액, 신용카드거래액, 체크카드거래액
- 목적: 비이자상품 확장 가능성이 있는 고객군 식별
- 결과: 고객 성장 사다리와 교차 확장 전략 설계

### 2. 이자이익 관점

- 핵심 변수: 원화대출잔액과 대출 상품군
- 목적: 안정적 수익 기여 고객과 상품별 잠재 타깃 식별
- 결과: 상품별 잠재고객 정의와 생애주기 관리 전략 제안

예금은 자금조달 측면에서는 중요하지만, 본 프로젝트의 핵심 질문인 `수익 구조 다변화`에는 직접성이 낮다고 판단해 핵심 세분화 기준에서는 제외했습니다.

## 주요 도전과 해결

### 1. 구간형 범주 데이터의 수치화

개인정보 보호 이슈로 인해 원시 수치가 아닌 구간형 범주 데이터가 제공되었습니다.  
예를 들어 대출잔액, 펀드잔액, 방카슈랑스잔액이 모두 `0~9구간` 형태로 주어졌고, 일부는 상한이 없는 개방구간이었습니다.

이 문제를 아래처럼 해결했습니다.

- 일반 구간은 구간 중앙값을 대표값으로 사용
- 개방구간은 단순 하한 대입 대신 분포 특성을 반영해 대표값 추정
- 동일 변수 내 치환 원칙의 일관성을 유지해 군집 해석 왜곡을 최소화

핵심은 "수치화" 자체보다, 수치화 과정에서 분석 원칙이 무너지지 않도록 한 것입니다.

### 2. 변수 중심이 아닌 문제 중심 분석 설계

변수가 많다고 모두 투입하면 군집 해석력이 오히려 떨어질 수 있다고 판단했습니다.  
그래서 팀은 먼저 BNK의 핵심 문제를 `이자이익 편중`과 `비이자이익 확장 정체`로 정의한 뒤, 그 문제를 가장 잘 설명하는 변수만 남겨 분석 구조를 설계했습니다.

이 결정이 이후 전략 제안의 해석 가능성을 크게 높였습니다.

## 내 역할

프로젝트에서 저는 **군집 결과를 실행 전략으로 구체화하는 역할**을 주로 맡았습니다.

- 각 군집의 특성을 정리하고 전략적으로 의미 있는 패턴 도출
- 비이자이익 관점의 성장 사다리 전략 설계
- 상품 간 조건부 확률을 활용한 교차 확장 전략 정리
- 이자이익 관점의 상품별 잠재고객 해석과 관리 방향 제안
- 최종 발표에서 바로 사용할 수 있도록 분석 결과를 문서형 스토리로 구조화

즉, 분석 결과를 "그래프 설명"에서 끝내지 않고, "그래서 누구에게 무엇을 해야 하는가"로 번역하는 역할에 집중했습니다.

## 전략 제안 요약

### 비이자이익 전략

- **성장 사다리 전략**: 하위 군집 고객이 상위 군집으로 이동하도록 단계별 진입 경로 설계
- **교차 확장 전략**: 카드, 예금, 펀드, 보험 간 연관 패턴을 활용해 거래 범위 확대
- **VIP 유지 전략**: 고기여 고객은 신규 판매보다 관계 유지와 맞춤 관리 우선

### 이자이익 전략

- **상품별 잠재고객 발굴**: 대출 상품별로 다른 고객 프로필을 기준으로 타깃 정의
- **생애주기 관리**: 연령대와 직업 특성에 맞춰 반복 수요가 발생하는 시점 관리
- **포트폴리오 관점 운영**: 단일 대출 전략이 아니라 상품군별 안정 수익 구조 설계

## 포트폴리오 관점에서 보여주는 역량

- 비즈니스 문제를 데이터 분석 프레임으로 전환하는 능력
- 제한된 데이터 조건에서도 해석 가능한 분석 구조를 설계하는 능력
- 군집화 결과를 실제 고객 관리 전략으로 연결하는 능력
- 해커톤 산출물을 GitHub 포트폴리오 문서로 재구성하는 능력
