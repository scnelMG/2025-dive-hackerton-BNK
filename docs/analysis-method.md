# 분석 방법

## 분석 목표

이 프로젝트의 목표는 BNK 고객을 단순 거래 규모가 아니라 `수익 기여 구조` 기준으로 해석하고, 각 고객군에 맞는 관리 전략을 도출하는 것이었습니다.

핵심 전제는 아래와 같습니다.

- 이자이익과 비이자이익은 수익 발생 방식이 다르다.
- 따라서 같은 고객 분류 체계로만 해석하면 전략적 의미가 약해질 수 있다.
- 고객 세분화는 통계적 분류로 끝나지 않고 실행 전략으로 이어져야 한다.

## 전체 분석 흐름

1. 문제 정의와 분석 축 설정
2. 구간형 데이터 전처리
3. 비이자이익 관점 군집화
4. 이자이익 관점 상품별 분석
5. 군집 해석과 전략 제안

## 1. 문제 정의와 분석 축 설정

분석은 `주어진 변수를 모두 쓰는 방식`이 아니라 `해결해야 할 사업 문제`에서 출발했습니다.

팀이 정의한 핵심 문제는 다음 두 가지였습니다.

- 이자이익 편중
- 비이자이익 확장 정체

이 문제를 가장 잘 설명하는 방식으로 분석 축을 분리했습니다.

### 비이자이익 트랙

- 목적: 비이자상품 확장 가능성이 높은 고객군 식별
- 중심 변수: 펀드잔액, 방카슈랑스잔액, 신용카드거래액, 체크카드거래액

### 이자이익 트랙

- 목적: 안정적 수익 기여 고객과 상품별 잠재 타깃 식별
- 중심 변수: 원화대출잔액, 대출 상품 유형, 고객 프로필

## 2. 구간형 데이터 전처리

제공 데이터는 개인정보 보호 이슈로 인해 원시 수치가 아니라 구간형 범주 데이터였습니다.  
따라서 군집화와 비교 분석을 위해 정량화 과정이 필요했습니다.

### 전처리 원칙

- 일반 구간은 구간 중앙값으로 치환
- 개방구간은 단순 하한 대입 대신 분포를 반영해 대표값 추정
- 동일 변수 안에서 치환 기준의 일관성 유지
- 총 대출 구간과 세부 대출 구간의 정합성 점검

이 과정의 목적은 모델 성능 향상보다, 이후 군집 해석이 왜곡되지 않도록 하는 것이었습니다.

## 3. 비이자이익 관점 군집화

비이자이익 분석은 고객의 비이자상품 이용 수준과 확장 가능성을 파악하는 데 초점을 맞췄습니다.

### 사용한 접근

- 구간값 수치화
- 로그 변환
- 스케일링
- K-Means 기반 군집화
- 적정 군집 수 검토
- 군집을 수익 기여 흐름에 맞게 재정렬

### 해석 방식

군집은 단순한 분류 결과가 아니라 `성장 단계`로 해석했습니다.

- Basic: 비이자상품 접점이 적은 초기 고객군
- Standard: 카드 등 생활거래 기반이 형성된 고객군
- Premium: 거래 심화가 진행된 고객군
- VIP: 다상품 이용과 높은 기여도를 보이는 핵심 고객군

이 구조를 바탕으로 성장 사다리 전략과 교차 확장 전략을 설계했습니다.

## 4. 이자이익 관점 상품별 분석

이자이익 분석은 대출잔액 규모 자체보다, 어떤 상품군에서 어떤 고객 특성이 나타나는지를 해석하는 방향으로 진행했습니다.

### 분석 의도

- 상품별 잠재고객을 구분
- 반복 수요가 발생하는 고객 특성을 확인
- 생애주기와 직업 특성에 맞는 관리 포인트 도출

### 해석 포인트

- 기업대출은 군집에 따라 직업 프로필이 다르게 나타남
- 서민전용대출은 특정 연령대와 핀테크 제휴대출 보유 패턴이 두드러짐
- 따라서 이자이익 전략은 단일 대출 전략보다 상품군별 관리 체계가 적합함

## 5. 결과 해석 원칙

이 프로젝트는 군집을 "보기 좋게 나누는 일"보다 "무엇을 실행할 것인가"에 맞춰 해석했습니다.

- 비이자이익 군집은 상향 전환 가능성 중심으로 해석
- 이자이익 군집은 상품별 관리 전략 중심으로 해석
- 최종 결과는 지표 요약보다 전략 언어로 정리
